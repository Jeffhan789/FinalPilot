# 02 落地执行方案

文档版本：v0.1  
日期：2026-05-01

## 1. 总体策略

FinalPilot 采用“先产品闭环，后技术增强”的开发策略。

第一阶段不追求所有 AI 功能都完全自动化，而是先完成一个稳定可演示的 iOS App：

课程 -> 计划 -> 测验 -> 诊断 -> 调整计划。

这样可以避免一开始被 PDF 解析、大模型调用、云端部署等复杂问题拖住，同时保留足够的技术深度用于项目展示。

## 2. 推荐技术栈

iOS 客户端：

- SwiftUI：构建界面。
- SwiftData：本地数据存储。
- Charts：展示学习分析图表。
- UserNotifications：发送复习提醒。
- Core ML：部署轻量掌握度预测模型。

云端服务：

- Supabase：认证、PostgreSQL 数据库、对象存储。
- Edge Functions：处理计划生成、诊断分析、AI 调用。

AI 与模型：

- 多智能体逻辑：后端服务中的多个 Agent 类。
- 神经网络模型：小型 MLP，用于预测知识点掌握度。
- Embedding：后期用于课程资料语义检索。

## 3. 版本路线

### v0.1 文档与原型阶段

目标：

- 明确需求。
- 明确页面结构。
- 明确技术架构。
- 准备知识点和题库种子数据。

产出：

- 需求规格说明书。
- 落地执行方案。
- 版面与交互设计文档。
- 技术架构文档。
- 数据模型与题库规划。
- 项目进展日志。

### v0.2 本地 MVP

目标：

- 不依赖云端，先做出可运行 App。
- 使用本地 JSON 或 SwiftData 保存课程、任务和题目。

功能：

- 首页 Dashboard。
- 课程列表。
- 知识点详情。
- 今日任务。
- 任务打卡。
- 测验练习。
- 简单薄弱点诊断。

验收标准：

- 可以完整演示一次复习闭环。
- 数据关闭 App 后不丢失。
- UI 基本成型。

### v0.3 智能计划与分析

目标：

- 加入更明确的计划生成算法。
- 加入图表分析和掌握度评分。

功能：

- 7 天计划自动生成。
- 优先级计算。
- 掌握度更新。
- 错题回顾。
- 学习分析页。

核心算法：

```text
priority_score =
  exam_urgency * 0.35 +
  weakness_score * 0.30 +
  difficulty * 0.20 +
  review_gap * 0.15
```

术语说明：

`priority_score` 是任务优先级分数。分数越高，越应该安排到更靠前的复习任务中。

### v0.4 云端同步

目标：

- 引入云计算能力。
- 实现用户数据跨设备保存。

功能：

- 用户登录。
- 课程数据云端同步。
- 答题记录云端同步。
- 计划生成结果云端保存。
- 云端 Agent API。

推荐 API：

```text
POST /generate-plan
POST /submit-quiz-attempt
GET /dashboard
GET /courses
GET /weakness-report
```

### v0.5 神经网络模型

目标：

- 让项目明确体现神经网络课程内容。
- 做一个轻量但完整的模型训练和部署流程。

模型任务：

预测某个知识点的掌握度。

输入特征：

- 最近正确率
- 复习次数
- 距离上次复习天数
- 平均答题时间
- 用户自信度
- 知识点难度

输出：

```text
mastery_score: 0.0 到 1.0
```

落地方式：

1. 用 Python 生成模拟学习数据。
2. 训练一个小型 MLP。
3. 导出为 Core ML 格式。
4. iOS 本地调用模型，更新知识点掌握度。

## 4. 六周执行计划

### 第 1 周：资料和原型

任务：

- 完成项目文档。
- 固化三门课知识点。
- 准备每门课 20 到 30 道题。
- 画出主要页面结构。

交付物：

- 文档 v0.1。
- 知识点 JSON。
- 题库 JSON。
- 页面流程图。

### 第 2 周：iOS 本地页面

任务：

- 创建 Xcode 项目。
- 开发首页。
- 开发课程页。
- 开发知识点页。
- 接入本地数据。

交付物：

- 可运行 App。
- 三门课展示完整。
- 基础导航完成。

### 第 3 周：任务和计划系统

任务：

- 实现 7 天计划生成。
- 实现今日任务。
- 实现任务打卡。
- 实现计划进度统计。

交付物：

- 可演示复习计划闭环。

### 第 4 周：测验和诊断

任务：

- 实现选择题和判断题。
- 实现评分。
- 实现错题记录。
- 实现薄弱点诊断。
- 根据测验结果调整知识点状态。

交付物：

- 可演示“答错 -> 诊断 -> 调整计划”。

### 第 5 周：云端与智能体

任务：

- 搭建 Supabase。
- 设计数据库表。
- 接入用户登录。
- 实现云端同步。
- 实现 Agent 服务接口。

交付物：

- 云端数据库。
- 后端 API。
- App 云同步演示。

### 第 6 周：模型与展示

任务：

- 训练掌握度预测模型。
- 转换 Core ML。
- 接入 iOS。
- 优化 UI。
- 编写展示文档。

交付物：

- 神经网络模块演示。
- 完整项目说明。
- 简历项目描述。

## 5. 风险与应对

风险：AI 功能太多，开发周期失控。  
应对：首版使用规则算法和预置数据，后续再接大模型。

风险：PDF/PPT 解析复杂。  
应对：首版不做自动解析，用结构化知识点数据替代。

风险：模型训练数据不足。  
应对：先使用模拟数据训练，重点展示完整流程。

风险：云端部署花费时间。  
应对：优先使用 Supabase，减少自建后端成本。
