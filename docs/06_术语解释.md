# 06 术语解释

文档版本：v0.1  
日期：2026-05-01

这个文件用于记录项目中出现的技术术语。后续我在推进项目时，会把关键术语持续补充进来，方便你审核和理解。

## MVP

MVP 是 Minimum Viable Product，中文常译为“最小可行产品”。

在本项目里，MVP 指的是先做一个功能不多但闭环完整的版本。FinalPilot 的 MVP 不追求所有 AI 功能都很复杂，但必须能完成：

课程设置 -> 计划生成 -> 测验 -> 薄弱点诊断 -> 计划调整。

## 多智能体

多智能体指多个具有不同职责的智能模块共同完成任务。

在 FinalPilot 中：

- PlannerAgent 负责复习计划。
- QuizAgent 负责题目推荐。
- DiagnosisAgent 负责薄弱点分析。
- CourseAgent 负责课程知识结构。
- CoachAgent 负责学习建议。

这些 Agent 不一定必须是独立服务器，首版可以是代码里的不同类或服务。

## Agent

Agent 可以理解为“有明确职责的智能角色”。

它通常包含：

- 输入信息
- 处理逻辑
- 输出结果

例如 DiagnosisAgent 的输入是答题记录，输出是薄弱点报告。

## 云计算

云计算指把计算、存储、数据库、认证等能力放在云端服务上，而不是全部放在用户手机里。

在 FinalPilot 中，云计算用于：

- 用户登录
- 云端数据库
- 多设备同步
- 文件存储
- 后端 Agent 服务

## Serverless

Serverless 是一种云端运行方式。开发者不需要自己管理服务器，只需要写函数，云平台负责运行。

在本项目中，可以用 Serverless 函数处理：

- 生成复习计划
- 分析薄弱点
- 同步用户记录

## Supabase

Supabase 是一个后端即服务平台，提供：

- 用户认证
- PostgreSQL 数据库
- 文件存储
- 云函数

它适合学生项目，因为开发速度快，功能完整。

## SwiftUI

SwiftUI 是苹果官方的 iOS 界面开发框架。

在本项目中，SwiftUI 用来开发：

- 首页
- 课程页
- 练习页
- 分析页
- 设置页

## SwiftData

SwiftData 是苹果提供的数据持久化框架，可以把 App 数据保存在手机本地。

在 FinalPilot 中，SwiftData 可以保存：

- 课程
- 知识点
- 今日任务
- 答题记录

## Core ML

Core ML 是苹果提供的机器学习模型部署框架。

在本项目中，Core ML 可以用来在 iPhone 本地运行掌握度预测模型。

## 神经网络

神经网络是一类机器学习模型，能够从数据中学习输入和输出之间的关系。

在 FinalPilot 中，神经网络用于预测某个知识点的掌握度。

## MLP

MLP 是 Multi-Layer Perceptron，多层感知机。

它是一种基础神经网络，适合处理结构化特征，例如：

- 正确率
- 复习次数
- 答题用时
- 自信度

## Embedding

Embedding 是把文本、图片或其他信息转换成向量的技术。

在后续版本中，FinalPilot 可以用 Embedding 判断：

- 用户笔记和知识点是否相关。
- 某道题对应哪个知识点。
- 哪些课程内容语义相似。

## 掌握度

掌握度是系统对用户理解某个知识点程度的估计。

在本项目中用 `mastery_score` 表示，范围是 0 到 1：

- 0 表示完全不会。
- 1 表示高度掌握。

## Exam Track

考试复习轨道。

在 FinalPilot 中，Exam Track 保存和安排所有与期末考试直接相关的任务。它拥有最高优先级。

## Career Track

校招面试轨道。

在 FinalPilot 中，Career Track 保存校招面试、笔试、项目讲解和简历复盘任务。它的优先级低于考试任务，但高于普通整理任务。

## ConflictAgent

冲突智能体。

它负责判断两个任务是否抢占同一段时间或同一部分精力，并决定任务是保留、压缩、合并还是延期。

## Career Minimum Pack

面试最低准备包。

这是考前阶段为了不让校招准备完全中断而设计的一组极简内容，通常包括自我介绍、项目介绍、技术亮点和常见问题回答。

## Fatigue Penalty

疲劳惩罚项。

当用户当天任务过多或连续高强度学习时，系统降低高消耗任务的优先级，避免计划看起来完美但现实中无法执行。
