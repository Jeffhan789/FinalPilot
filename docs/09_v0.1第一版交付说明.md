文档版本：v0.1  
日期：2026-05-01  
交付类型：iOS SwiftUI 本地 MVP

## 1. 本版目标

v0.1 的目标不是一次性完成云端、模型和正式上架，而是先交付一个可以打开、可以审核、可以继续迭代的 iOS App 骨架。

本版已经把前期方案落成代码：

- 考试复习优先。
- 机动事务保温。
- 双轨任务调度。
- 课程掌握度展示。
- 主动回忆练习。
- 答题后更新掌握度。
- 答错后自动加入复盘任务。
- 分析页展示风险和负载。

## 3. 已实现核心代码结构

```text
FinalPilotApp/
├── FinalPilotApp.swift
├── ContentView.swift
├── Models.swift
├── SeedData.swift
├── AppStore.swift
├── Theme.swift
├── Components.swift
├── DashboardView.swift
├── CoursesView.swift
├── PracticeView.swift
├── AnalyticsView.swift
└── CareerView.swift
```

代码职责：

- `Models.swift`：课程、知识点、任务、题目、设计复盘事件等数据模型。
- `SeedData.swift`：首版内置课程、知识点、题目和任务。
- `AppStore.swift`：本地状态管理、答题判分、掌握度更新、复盘任务生成。
- `Theme.swift`：颜色和视觉规范。
- `Components.swift`：复用 UI 组件。

## 4. 当前验证结果

已通过 generic iOS device 构建：

```text
xcodebuild -project FinalPilotApp.xcodeproj \
  -scheme FinalPilotApp \
  -configuration Debug \
  -sdk iphoneos \
  -destination 'generic/platform=iOS' \
  -derivedDataPath /private/tmp/FinalPilotDeviceDerived \
  CODE_SIGNING_ALLOWED=NO \
  build
```

结果：

```text
BUILD SUCCEEDED
```

说明：

- 当前环境没有可用 iOS Simulator runtime，所以模拟器构建会在 asset/simulator runtime 阶段失败。
- 已用 generic iOS device 构建验证工程和 Swift 源码可以编译。
- 为了避免当前环境的 asset catalog 编译问题，v0.1 暂未把 `Assets.xcassets` 放入 Xcode 构建阶段。

## 5. 本版限制

v0.1 暂不包含：

- 用户登录。
- 云端同步。
- SwiftData 持久化。
- Core ML 掌握度预测模型。
- 课程资料上传。
- AI 自动生成题目。
- 正式 App Icon。

这些功能会在后续版本逐步接入。

## 6. 审核重点

建议先审核以下问题：

1. 首页双轨冲刺指挥台是否符合真实复习压力。
2. Exam Track 和 Flex Track 的优先级表达是否清楚。
3. 里程碑最低准备包是否足够实用。
4. 练习页的“答题 -> 掌握度更新 -> 复盘任务”闭环是否合理。
5. 分析页指标是否能帮助你判断今天该做什么。

## 7. 下一版建议

v0.2 建议优先做：

- SwiftData 本地持久化。
- 任务编辑和自定义考试日期。
- 更多题库数据。
- App Icon 和 Launch Screen。
- 在本机 Xcode 模拟器中做视觉检查。
- 根据审核意见调整首页信息密度。
