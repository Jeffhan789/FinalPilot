# 03 版面与交互设计

文档版本：v0.1  
日期：2026-05-01

## 1. 设计原则

FinalPilot 是复习工具，不是营销页面。整体设计应当安静、清晰、可快速扫描。

设计关键词：

- 专注
- 高效
- 轻量
- 学术感
- 数据可视化

避免：

- 大面积装饰性渐变。
- 过多卡片嵌套。
- 只好看但不利于复习的布局。
- 首页出现无关介绍性文案。

## 2. 信息架构

底部 Tab 建议使用 5 个入口：

```text
今日
课程
练习
分析
校招
```

每个入口职责：

- 今日：告诉用户现在该做什么。
- 课程：展示三门课和知识点状态。
- 练习：进入题目、错题和模拟测验。
- 分析：展示进度、薄弱点和掌握度预测。
- 校招：管理面试事件、项目讲解卡片和最低准备包。

## 3. 首页设计

首页名称：今日

核心目标：

用户打开 App 后 10 秒内知道今天的复习安排。

页面模块：

1. 顶部状态区
   - 今日日期
   - 最近考试倒计时
   - 今日推荐学习时长

2. Exam Track 区
   - 显示今日必须完成的考试复习任务
   - 按 Must / Should / Skip 分层

3. Career Track 区
   - 显示今日最低限度面试准备
   - 默认不超过 15 到 30 分钟
   - 当考试风险高时自动压缩或延期

4. 今日任务区
   - 按优先级排列任务
   - 每个任务显示课程、知识点、时长、任务类型
   - 支持完成打卡

5. 风险提示区
   - 显示 1 到 3 个高风险薄弱点
   - 点击后进入对应知识点详情

6. 三门课进度区
   - 每门课一个横向进度条
   - 显示掌握度、考试倒计时、待复习任务数

视觉建议：

- 背景使用浅灰白。
- 主要操作按钮使用稳重蓝绿色。
- 风险提示使用柔和橙色或红色，不使用刺眼警告色。

## 4. 课程页设计

页面名称：课程

展示内容：

- 多智能体系统
- 云计算
- 神经网络

每个课程条目展示：

- 课程名
- 考试倒计时
- 已掌握知识点数量
- 薄弱知识点数量
- 总体进度条

进入课程详情后展示：

- 章节列表
- 知识点列表
- 每个知识点的掌握度
- 最近练习结果
- 推荐下一步动作

知识点状态视觉：

- 未开始：灰色圆点
- 复习中：蓝色圆点
- 已掌握：绿色圆点
- 薄弱：橙色圆点

## 5. 练习页设计

页面名称：练习

主要功能：

- 今日推荐练习
- 按课程练习
- 错题回顾
- 模拟考试

答题界面：

- 顶部显示课程和知识点。
- 中部显示题目。
- 选项使用清晰按钮列表。
- 底部显示提交按钮。
- 提交后展示正确答案和解释。

用户完成题目后记录自信度：

```text
不确定 / 一般 / 很确定
```

术语说明：

自信度是用户主观判断自己是否真正理解知识点。它可以辅助系统区分“蒙对”和“真正掌握”。

## 6. 分析页设计

页面名称：分析

页面模块：

1. 总览指标
   - 总复习时长
   - 总完成任务
   - 平均正确率
   - 高风险知识点

2. 课程掌握度
   - 三门课进度对比

3. 薄弱点列表
   - 按风险从高到低排序
   - 每个薄弱点给出建议动作

4. 学习趋势
   - 最近 7 天任务完成率
   - 最近 7 天正确率

## 7. 校招页设计

页面名称：校招

功能：

- 面试时间线
- 公司和岗位列表
- 面试轮次状态
- Career Minimum Pack
- FinalPilot 项目讲解卡片
- 常见面试问题练习

设计原则：

- 考前阶段不做复杂招聘管理系统。
- 页面只保留“下一场面试”和“最低准备任务”。
- 当考试距离小于 3 天时，校招页提示用户只保留必要准备。

## 8. 视觉规范

推荐色彩：

- 主色：深青蓝 `#256D75`
- 强调色：柔和橙 `#E98B3A`
- 成功色：绿色 `#2F9E66`
- 文本主色：近黑 `#1F2933`
- 辅助文本：灰色 `#6B7280`
- 背景：冷白灰 `#F6F8FA`

字体层级：

- 页面标题：22 到 28 pt
- 模块标题：16 到 18 pt
- 正文：14 到 16 pt
- 辅助说明：12 到 13 pt

组件规范：

- 卡片圆角不超过 8 pt。
- 图标优先使用 SF Symbols。
- 任务列表使用紧凑布局，避免过度留白。
- 进度条高度 6 到 8 pt。
- 按钮文字不超过 6 个汉字为佳。

## 9. 主要用户流程

### 流程 A：每日复习

```text
打开 App
-> 查看今日任务
-> 进入第一个任务
-> 完成复习或测验
-> 打卡完成
-> 系统更新进度
```

### 流程 B：测验诊断

```text
进入练习页
-> 选择神经网络课程
-> 完成反向传播题目
-> 系统判分
-> 记录自信度
-> 生成薄弱点诊断
-> 调整明日计划
```

### 流程 C：考前冲刺

```text
进入分析页
-> 查看高风险知识点
-> 点击薄弱点
-> 进入错题回顾
-> 完成追加练习
-> 掌握度重新计算
```
