# 12 固定路径实时同步方案

文档版本：v0.1  
日期：2026-05-01

## 1. 结论

可以实现从固定路径同步进度，但不能让 iOS App 直接读取 Mac 上的 `/Users/...` 路径。

原因是 iOS 有沙盒限制：真机或模拟器中的 App 默认只能访问自己的容器、用户明确选择的文件、网络接口和打包资源，不能直接扫 Mac 的任意文件夹。

因此当前采用 `Local Sync Bridge` 方案：

```text
三条固定 iCloud 路径
-> Mac 本地同步脚本扫描
-> 生成 StudySyncSnapshot.json
-> App 计划页读取快照
-> 开启本地服务时可点击刷新获取最新数据
```

## 2. 固定同步路径

当前同步器读取三条用户指定的本机固定路径：

- C310 多智能体目录
- E320 神经网络目录
- C310_E320 高效复习总控目录

真实绝对路径保存在本机文件 `tools/sync_sources.local.json`，该文件已加入 `.gitignore`，不会推送到 GitHub。仓库中只保留 `tools/sync_sources.example.json` 作为配置模板。

## 3. 挑选规则

同步器不是把所有课件都塞进 App，而是按文件名和目录优先级挑选“进度数据”：

- 每日任务看板
- README 总计划
- 每日具体任务安排
- A4 两周规划表
- 执行清单
- 真题矩阵
- 错题、复盘、错因
- 打印清单与笔记规范

普通课件 PDF 只作为低优先级元数据，不进入首页任务流。

## 4. 使用方式

生成一次内置快照：

```text
node tools/sync_study_sources.mjs --write
```

这会同时生成两类文件：

- `StudySyncSnapshot.json`：脱敏版本，可进入 GitHub。
- `StudySyncSnapshot.local.json`：本机私有版本，包含真实路径和摘要，已加入 `.gitignore`，不推送。

开启实时服务：

```text
node tools/sync_study_sources.mjs --serve
```

然后在 App 中：

1. 打开 `计划` 页。
2. 找到 `固定路径同步`。
3. 点击 `刷新`。
4. 如果服务开着，显示 `实时服务`；否则显示 `内置快照`。

注意：`实时服务` 返回的是本机私有完整快照，只适合本机调试和演示，不提交到 GitHub。

## 5. 后续升级

下一步可以把这个同步桥升级为真正的云同步：

- Mac 端扫描固定路径后上传到 CloudKit / Firebase。
- iPhone 真机从云端拉取进度快照。
- App 内勾选完成后回写云端。
- 同步器再把完成状态写回 Markdown 看板。
